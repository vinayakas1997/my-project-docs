Industrial Edge Data Platform

End-to-end factory data platform on LINX TRITON edge controllers — multi-path PLC ingestion, n8n + FastAPI processing into PostgreSQL, Grafana analytics, and Ansible Semaphore fleet management.

CompanySomic Ishikawa
RoleR&D Engineer
Year2024 — Present
TypePlatform / Infrastructure
IIoT

Overview

The factory already ran IoT on the floor, but the stack depended on commercial software licenses and a capped register count — limiting how much granular shop-floor data could be exposed for analysis. This project replaced that constraint with a self-owned edge-to-database pipeline, improving cost efficiency and data accessibility without replacing Omron PLC hardware.

Modernized factory data at Somic Ishikawa by standardizing edge access on OPC UA and moving signals through a structured pipeline to queryable analytics for production teams.

Edge compute runs on TRITON controllers from LINX (CODESYS, native OPC UA), with a Python FINS bridge running on-device. Raw data is ingested over three paths (OPC UA, MQTT, CSV) matched to latency and volume, orchestrated by n8n, processed and persisted by FastAPI → PostgreSQL, and visualized in Grafana. Ansible Semaphore manages the TRITON fleet; Prometheus monitors pipeline health and logs.

This platform feeds Part 2: Agentic Factory Analytics — AI and agentic analysis on the same PostgreSQL data layer. Full case study expanding soon.

Ingestion Strategy

Each protocol path was chosen by data profile — not every signal uses the same transport.

Path Use case Rationale
OPC UA Standardized edge access + lightweight live streams IIoT interoperability on TRITON; HTTP used within OPC UA for real-time feeds
MQTT Large, fast live datasets Pub/sub for high-throughput tags
CSV Slow / batch data Efficient infrequent or bulk loads

Key Achievements

  • Replaced license- and register-limited legacy IoT with a self-owned edge-to-PostgreSQL pipeline, reducing vendor dependency and cost
  • Built a Python FINS bridge for Omron PLCs — no PLC hardware replacement required
  • Deployed LINX TRITON edge controllers as the OPC UA aggregation layer
  • Designed multi-path ingestion (OPC UA, MQTT, CSV) matched to data velocity and volume
  • Implemented n8n workflows and a FastAPI processing node → PostgreSQL as system of record
  • Delivered Grafana dashboards querying PostgreSQL for production analytics
  • Unlocked higher-granularity factory data for the agentic AI team — Part 2: Agentic Factory Analytics
  • Introduced Ansible Semaphore for repeatable TRITON fleet provisioning and configuration
  • Used Prometheus for pipeline and log health monitoring with failure detection
