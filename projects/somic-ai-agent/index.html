CompanySomic Ishikawa
RoleR&D Engineer
Year2024 — Present
TypeRAG / AI Agent

Overview

At Somic Ishikawa, internal technical knowledge is spread across documents that are difficult to search quickly. This project builds a RAG retrieval agent so teams can get grounded answers from internal sources instead of relying on guesswork or manual lookup.

The RAG pipeline, document ingestion, and agent behavior are in progress. Architecture, tech stack, and measured outcomes will be published here as the case study is completed.

Architecture

Hybrid RAG in three stages: MinerU extraction with hallucination-checked retries, GLM-4.6V VLM fallback for diagram pages, cleaning, and PostgreSQL raw/clean document stores before indexing into Hindsight (world facts, observations, experience), hybrid retrieval (RRF, semantic, BM25), and local vLLM inference on DGX Spark.
