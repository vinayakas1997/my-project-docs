Overview

Phase 2 of the Intelligent RAG platform at Somic Ishikawa — extending the autonomous ingestion pipeline into an Enterprise AI Platform with two major additions: a Human-in-the-Loop (HITL) Evaluation Dashboard and a specialized PLC ingestion pipeline with entity-anchored metadata enrichment.

Part 1 solved the core problem of making 2,500+ bilingual documents queryable in seconds. Phase 2 solves two new problems: (1) enabling non-technical business teams to self-serve regression testing of model updates, and (2) expanding ingestion to handle PLC automation manuals with hyper-specific technical registers and memory addresses — a fundamentally different chunking and retrieval challenge.

Production Context & Business Impact

The Eval Bottleneck: Whenever the team updated the underlying model or retrieval pipeline, business users had to request engineering to manually run regression scripts. The automated LLM-as-a-judge evaluation suffered from high False Negative rates — marking semantically accurate answers as "Failed" because phrasing differed from the golden answer.

The PLC Expansion: The PLC Code Design team requested access to the platform for 500-page automation manuals. These documents are dense with hardware registers (%MW100), error codes, Kanban identifiers, and network addresses. Standard text chunking shattered the structural relationship between register addresses and their functional descriptions — destroying retrieval accuracy.

The Solution: A self-serve Golden Dataset Management Dashboard with HITL override capability for eval, and a Metadata Enrichment Pipeline that extracts technical entities before chunking and binds them to vector metadata payloads for hybrid filtering.

Measurable Impact: Non-technical teams gained independent eval capability. PLC retrieval accuracy reached 85-90% on complex register-level queries.

Core Engineering Challenges & Solutions

1. Human-in-the-Loop Evaluation Dashboard

The Bottleneck: Initially, every model update required engineering to manually run regression scripts for the business team. The automated LLM-as-a-judge evaluation, while useful, suffered from excessive False Negatives — marking correct answers as failed when vocabulary or phrasing differed from the golden ground truth.

The Architecture: Built a localized Golden Dataset Management Dashboard where non-technical team leads can input, edit, and version their own ground-truth question-answer pairs. The system automates regression evaluation via an LLM judge scoring both correctness and source attribution.

The Solution: Integrated a Human-in-the-Loop (HITL) Override System. The AI judge acts as a first-pass filter; domain experts can audit flagged failures, override the machine verdict, and re-mark cases as passed. This produces a clean, enterprise-audited benchmarking dataset over time — eliminating the false-negative bottleneck without sacrificing rigor.

2. Specialized PLC Ingestion Pipeline

The Expansion: Scaled the platform to serve the PLC Code Design team, who needed to query 500-page automation manuals dense with hardware registers (%MW100), memory addresses, Kanban identifiers, error codes, and network protocols.

The Challenge: Standard text chunking (e.g., 500-token sliding windows) shatters the structural relationship between a register address and its functional description. When a PLC engineer queries a specific register, the vector store returns chunks that mention the register but lose the surrounding context — destroying retrieval accuracy.

The Solution (Hindsight Tagging): Developed a pre-ingestion Metadata Enrichment Pipeline. Before chunking, a targeted LLM scans each document to extract hyper-specific technical primitives: register addresses, error codes, protocol identifiers, and system names. These extracted entities are hard-bound to each vector chunk's Metadata Payload. When a PLC engineer queries a register address, the system uses Hybrid Filtering — Exact Metadata Match combined with Vector Similarity — instead of relying solely on noisy semantic similarity.

The Result: Retrieval accuracy on complex technical queries rose to 85-90%, enabling the PLC team to find register-level information in seconds instead of manually flipping through hundreds of pages.

Key Achievements & Impact

  • Delivered self-serve Golden Dataset Management Dashboard — non-technical teams manage ground-truth QA pairs, trigger regression eval runs, and inspect results independently
  • Built HITL Override System — domain experts audit and override LLM-as-judge false negatives, producing a clean enterprise-audited benchmark over time
  • Developed Entity-Anchored Ingestion (Hindsight Tagging) — pre-chunking metadata enrichment extracts registers, error codes, and protocol IDs, bound to vector payloads
  • Implemented Hybrid Filtering (Exact Metadata Match + Semantic Similarity) — PLC query accuracy improved from ~45% to 85-90%

  • Eliminated the engineering eval bottleneck — business teams now self-serve regression testing without developer intervention
  • All systems remain fully on-premise — no external APIs, data never leaves the local DGX Spark environment
